# AWS Infrastructure Import Configuration

This Terraform configuration imports 93 existing AWS resources across 17 regions into Terraform state management.

## Resources Imported

- **17 VPCs** - Default VPCs across multiple AWS regions
- **55 Subnets** - Default subnets in various availability zones
- **19 Security Groups** - Default and custom security groups
- **1 EC2 Instance** - Test instance in us-east-1

## Prerequisites

1. Terraform >= 1.5.0
2. AWS CLI configured with appropriate credentials
3. Access to AWS account 438180981611
4. Appropriate IAM permissions to read and import resources

## Usage

### Step 1: Initialize Terraform

bash
terraform init


### Step 2: Review the Import Plan

bash
terraform plan


### Step 3: Apply the Configuration

This will import all resources into Terraform state:

bash
terraform apply


### Step 4: Verify Imports

Check that all resources were imported successfully:

bash
terraform state list


## File Structure

- `provider.tf` - AWS provider configuration with region aliases
- `variables.tf` - Variable definitions
- `terraform.tfvars` - Variable values (includes conversation_id)
- `main.tf` - Resource definitions using for_each patterns
- `imports.tf` - Import blocks for all resources (Terraform 1.5+ syntax)
- `outputs.tf` - Useful outputs organized by resource type and region

## Important Notes

### Tags

- **No additional tags are added** - only existing tags are preserved
- The EC2 instance has one tag: `Name = "test"`
- All other resources have no tags
- **No default_tags** in provider configuration

### Security Groups

- Security group rules are managed separately using `aws_security_group_rule` resources
- Default security groups allow ingress from self and egress to all
- Custom security groups (launch-wizard-1, launch-wizard-2) allow SSH access

### Multi-Region Configuration

- Uses provider aliases for each region where resources exist
- Resources are organized using locals with region mappings
- Outputs are available by region, VPC, and resource type

## Outputs

After applying, you can view resource information:

bash
# View all VPC IDs
terraform output vpc_ids

# View VPCs by region
terraform output vpc_ids_by_region

# View resource summary
terraform output resource_summary


## Modifying Resources

After import, you can modify resources as needed. Remember:

1. Changes to default resources (VPCs, subnets, security groups) may affect other AWS services
2. Always run `terraform plan` before applying changes
3. Consider using `-target` flag for selective updates

## Removing from Management

To remove a resource from Terraform management without deleting it:

bash
terraform state rm <resource_address>


Example:
bash
terraform state rm 'aws_vpc.imported["vpc-0dcb8ae2b13fcace7"]'
